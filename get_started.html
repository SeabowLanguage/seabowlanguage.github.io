Seabow data types:

Data types are used to store many kind of data in different form. Here all the primitive types of seabow (i.e basic types).

Type Description Memory
byte a signed 8-bit integer 1o (-128 ↔ 127)
ubyte an unsigned 8-bit integer 1o (0 ↔ 255)
short a signed 16-bit integer 2o (-32,768 ↔ 32,767)
ushort an unsigned 16-bit integer 2o (0 ↔ 65,535)
int a signed 32-bit integer 4o (-2,147,483,648 ↔ 2,147,483,647)
uint an unsigned 32-bit integer 4o (0 ↔ 4,294,967,295)
long a signed 64-bit integer 8o (-9,223,372,036,854,775,808 ↔ 9,223,372,036,854,775,808)
ulong an unsigned 64-bit integer 8o (0 ↔ 18,446,744,073,709,551,615)
int128 a signed 128-bit integer 16o (≈ -1,701 x 10³⁸ ↔ ≈ 1,701 x 10³⁸)
uint128 an unsigned 128-bit integer 16o (0 ↔ ≈ 3,403 x 10³⁸)
float a 32-bit decimal number 4o (6 decimal digits)
double a 64-bit decimal number 8o (15 decimal digits)
ldouble a 128-bit decimal number 16o (18 decimal digits)
bool a boolean value 1o (true or false)
char a UTF-32 character 4o (any simple character)
string a sequence of UTF-8 characters 1o x number of characters ("abc", "平仮名, ひらがな")
error a seabow error that notices an unexpected thing happened store an error name, details, error's line and error's column
type type of a seabow value 1o (<byte>, <string>, ...)
any represent a seabow value, regardless of type size of the seabow value
array a sequence of seabow values of a specific type size of all seabow values stored
list a resizable sequence of seabow values of a specific type size of all seabow values stored
dict a resizable sequence of key-value relation of specific types size of all key-value stored
function a sequence of seabow line-code with arguments and returned values

Variables in seabow:

In seabow, you can declare variables which allows you to store values like this:

<modifiers>  
<type>  
<var_name>
;
<modifiers>  
<type>  
<var_name>  
=  
<value>
;

<modifiers> ➜ Optional. Used to modify usability of the variable.

  • const: used to specify that the variable can not be reassigned.
  • global: Used to specify that the variable can be used everywhere on the code.

<type> ➜ The type of data you want to store in the variable.

<var_name> ➜ The name you want to give to the variable.

<value> ➜ Optional. The value to store in the variable.
